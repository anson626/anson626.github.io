amazon 密码设定

2022-09-04 1 min read

所有tidb相关主机 检查 ssm服务是否启动

检查命令

  sudo systemctl status amazon-ssm-agent

如果没有启动 执行

yum install -y https://s3.ap-east-1.amazonaws.com/amazon-ssm-ap-east-1/latest/linux_amd64/amazon-ssm-agent.rpm && \
systemctl enable amazon-ssm-agent && \
systemctl start amazon-ssm-agent && \
systemctl status amazon-ssm-agent

所有tidb相关主机修改用户配置

所有主机修改ssh配置 /etc/ssh/sshd_config 中的 #PasswordAuthentication yes 修改 PasswordAuthentication yes （取消注释）

修改配置后重启 sshd

systemctl restart sshd

所有主机修改用户密码

需要修改root 和 tidb 两个用户

#修改密码方式

passwd [userid]
#随后输入两次新密码即可

示例 ：passwd root

将所有tidb主机添加到新的jumpserver

要求 dba 链接堡垒机 只能使用tidb用户登录 节点

  • 所有tidb相关主机 检查 ssm服务是否启动
    • 检查命令
    • 如果没有启动 执行
  • 所有tidb相关主机修改用户配置
  • 所有主机修改用户密码
  • 将所有tidb主机添加到新的jumpserver
