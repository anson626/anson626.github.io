git init就是git库初始化的命令。如你所说，在任何一个文件夹下执行就是初始化这个文件夹，
此时执行git status，可以看到文件夹下的所有文件都变成未被跟踪的文件。
git init

初始化后，需要先注册
git config --global user.name ‘anson’
git config --global user.email ‘ansonyean626@gmail.com’

添加项目到临时工作区
git add index.html


git status // 可以看到文件的状态

git add 《文件》   // 添加单一文件测试环境
git rm --cached index.html 《文件》  //在测试环境中移除

Git 远程

查看远程的提交日志信息 （仓库名/分支）
git log origin/main

显示远程的详细信息
git branch -va 

设定远程邮件
git config --global user.name anson
git config --global user.email anson@testing.com

添加指定的文件到缓存区
[root@test edisguns]# git add README.md 

保存到 本地区
[root@test edisguns]# git commit -m "Readme.md"

推送到远程机器
git push origin --all

取origin 仓库的分支
git fetch origin devlop

切换分支
git checkout devlop

合并
git merge FETCH_HEAD

本地删除
git branch -D devlop
删除后，再远程推一次，就可以远程删除了
git push origin :devlop


打标签
git tag "v0.2"
把已打好的标签一起推送上去
git push origin --tags

Github 推送方式

#需要到setting 》 developer 》 personal token 上设定token先
git remote set-url origin https://<GITHUB_ACCESS_TOKEN>@github.com/<GITHUB_USERNAME>/<REPOSITORY_NAME>.git
git push origin main
