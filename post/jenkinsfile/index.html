目标服务器 - Java服务规划

  • 路径规划
#创建个服务目录 jar包存放在此目录
mkdir -p /usr/local/src/java/gun
chown -R java-app:devops /usr/local/src/java
chmod -R 770 /usr/local/src/java
  • 生成 unitfile
cat >/usr/lib/systemd/system/guns.service <<'EOF'
[Unit]
Description=Manage Java service
[Service]
WorkingDirectory=/usr/local/src/java/guns
Environment=SERVER_PORT=9001
Environment=JAR_NAME=guns.jar
ExecStart=/bin/bash -c "exec /usr/bin/java \
  -Dlog4j2.formatMsgNoLookups=true \
  -jar $JAR_NAME --server.port=$SERVER_PORT  \
  >> output.log 2>&1 "
User=java-app
Type=simple
Restart=on-failure
RestartSec=10
[Install]
WantedBy=multi-user.target
EOF

安装jenkins 插件

安装2个常用的jenkins插件

  • Extended Choice Parameter

    配上 update，restart，stop，rollback

  • Git Parameter

    安装后需要会显示一些报错，只要执行拉取一次后参数就会出来了

Jenkins File

  • 配置参数 如 - Git Parameter等等 - http://www.mydlq.club/article/45/
  • 使用pipeline模块
pipeline {
  agent any
  tools {
    //名称是根据maven 全局配置的
    maven 'maven3.6.3'
  }
  environment {
	GIT_URL="git@35.75.19.227:root/edisguns.git"
	KEY_DIR="/var/lib/jenkins/key/id_rsa"
  }

  stages {
    stage('拉代码') {
		//Git Parameter 需要执行一次才能看到选项
        steps {
            checkout([$class: 'GitSCM', branches: [[name: "${BRANCH}"]], extensions: [], userRemoteConfigs: [[credentialsId: 'ssh-git-key-guns', url: "${GIT_URL}"]]])
        }
    }
    stage('编译代码') {
        steps {
            sh """
            mvn clean install -Dmaven.test.skip=true
            """
        }
    }
	
	stage('更新包到主机运行') {
		steps {
			sh """
				echo "======开始复制包到目标机器======"
                scp -i ${KEY_DIR} -o StrictHostKeyChecking=no -P 22 target/guns.jar devops@test.guns:/usr/local/src/java/guns/ && \
                ssh -i ${KEY_DIR} -o StrictHostKeyChecking=no -p 22 devops@test.guns  "sudo systemctl restart guns.service"
			"""
			}
	}
  }
}

For 循环

  • guns_service 变量 是使用 Extended Choice Parameter 变量

    注意： delimiter 是需要修改为空格

pipeline {
  agent any

  stages {
	stage('Test') {
		steps {
			sh """
				for INT in ${guns_service};
				do
					echo "\${INT}"
				done
			"""
			}
	}
  }
}

Parallel 并行构建

  • https://blog.csdn.net/weixin_31323635/article/details/105118824
  • 参考例子，
pipeline {
  agent any

  environment {
	GIT_URL="git@35.75.19.227:root/edisguns.git"
	KEY_DIR="/var/lib/jenkins/key/id_rsa"
  }
  stages {
	stage('更新包到主机运行') {
		parallel {
			stage('guns') {
				steps {
					sh """
							echo "======开始复制包到目标机器======"
							ssh -i ${KEY_DIR} -o StrictHostKeyChecking=no -p 22 devops@test.guns  "sudo systemctl restart guns.service"
					"""
				}
			}
			stage('guns2') {
				steps {
					sh """
							echo "======开始复制包到目标机器guns2======"
							ssh -i ${KEY_DIR} -o StrictHostKeyChecking=no -p 22 devops@test.guns  "sudo systemctl restart guns2.service"
					"""
				}
			}
			stage('guns3') {
				steps {
					sh """
							echo "======开始复制包到目标机器guns3======"
							ssh -i ${KEY_DIR} -o StrictHostKeyChecking=no -p 22 devops@test.guns  "sudo systemctl restart guns3.service"
					"""
				}
			}
		}
	}
  }
}
